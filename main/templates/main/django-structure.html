2. Key Files & Their Roles

  • manage.py: Project command-line tool ‚Äî runserver, migrate, etc.
  • settings.py: All configuration ‚Äî installed apps, DB setup, static files.
  • urls.py: Maps URLs to views.
  • wsgi.py / asgi.py: Used for production deployment.

4. Best Practices

  • Keep each feature in a separate app.
  • Use environment variables for sensitive data.
  • Follow consistent folder structure and naming.
