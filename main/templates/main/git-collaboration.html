GitHub Collaboration

Pull Requests

Pull Requests (PRs) are how developers review and propose changes. They allow teams to review, discuss, and merge code into the main branch.

üí° Always assign reviewers and link related issues in your PR.

Issues

Issues help track bugs, feature requests, and tasks. You can label, assign, and reference them in PRs.

#123 Adds login feature linked to Issue #45

Project Boards

Project boards organize issues and pull requests into columns like To Do, In Progress, and Done. This gives a Kanban-style view of your project progress.

Forking & Contributions

For open-source contributions, users fork a repo, make changes, and create a PR back to the original project.

git clone https://github.com/username/repo.git
git checkout -b feature-branch
# Make changes
git push origin feature-branch
