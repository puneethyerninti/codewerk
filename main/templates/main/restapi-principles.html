RESTful Principles

What is REST?

REST (Representational State Transfer) is an architectural style for building APIs. It defines a set of constraints to make APIs scalable, stateless, and easy to understand.

The 6 REST Principles

  • 1. Client-Server: Frontend (client) and backend (server) should be independent.
  • 2. Stateless: Each API call should contain all information needed. No session state is stored on the server.
  • 3. Cacheable: Responses should define if they can be cached to improve performance.
  • 4. Uniform Interface: The structure of API responses should be consistent and standardized (usually JSON).
  • 5. Layered System: Clients shouldn't know whether they're talking to the actual server or an intermediate proxy.
  • 6. Code on Demand (optional): Servers can send executable code to clients (rarely used).
