Relational Database Design

What is Relational Design?

Relational design structures data into tables (relations), where each row is a record and each column is an attribute (field).

üí° Tip: Every table should have a unique primary key!

Primary & Foreign Keys

Primary Key: Uniquely identifies each row in a table.
Foreign Key: A field that links one table to another.

Types of Relationships

  • One-to-One (1:1)
  • One-to-Many (1:M)
  • Many-to-Many (M:N)

Use junction tables to handle many-to-many relationships.

Normalization

Normalization removes redundancy and improves integrity:

  • 1NF: Atomic columns
  • 2NF: Remove partial dependencies
  • 3NF: Remove transitive dependencies
