Props & State in React

What are Props?

Props (short for properties) are how data is passed from one component to another in React. They are read-only and help components become reusable.

function Welcome(props) {
  return <h1>Hello, {props.name}!</h1>;
}

Here, props.name accesses the value passed to the component.

Passing Props

Props are passed to components like attributes in HTML.

<Welcome name="Puneeth" />
üí° Tip: Props can be strings, numbers, objects, or even functions!

What is State?

State is a built-in object in React components that allows them to store dynamic data that can change over time.

import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);
  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>+</button>
    </div>
  );
}

Props vs State

  • Props: Passed from parent to child. Immutable.
  • State: Managed within the component. Mutable using useState().
